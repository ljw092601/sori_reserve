Problem

예약이 네이버 카페 댓글로 돌아갔습니다

  • 임원 한 명이 네이버 카페에 예약 글을 매번 직접 작성
  • 팀원들은 그 글에 원하는 시간을 댓글로 신청
  • 겹치는지 보려면 댓글을 일일이 대조 — 현황이 한눈에 안 보임
목표 → 글 작성도, 댓글 취합도 없이
현황을 한눈에 보고, 겹치면 애초에 예약이 안 되게.

기존 방식 — 실제 카페 글

↻ 글 하나에 댓글로 시간 신청 — 매번 반복

Core Design · 1

이중 예약 방지 — DB가 원자적으로 차단

-- 시간 범위가 겹치면(&&) 삽입 자체를 거부
ALTER TABLE reservations
  ADD CONSTRAINT no_overlap
  EXCLUDE USING gist (
    tstzrange(starts_at, ends_at) WITH &&
  );

-- 충돌 시 PostgreSQL 에러코드
-- 23P01 → HTTP 409 Conflict 로 매핑
  • 앱이 아니라 DB 차원에서 겹침을 차단
  • 동시 요청이 와도 원자적으로 한 건만 성공
  • 제약 위반 23P01 → HTTP 409
"겹치나 먼저 확인" 로직 없이도
레이스 컨디션이 구조적으로 불가능합니다.
Core Design · 2

타임존 버그 없이 KST 고정

  • Vercel 서버는 UTC로 동작 → 그대로 두면 9시간 밀림
  • 고정 +09:00 오프셋 헬퍼로 항상 한국 시간 기준 처리
  • 주간 그리드 · 예약 저장 · 금지 규칙 모두 동일 기준
// 서버 TZ와 무관하게 KST 고정
const KST_OFFSET = "+09:00";

function toKstRange(date, h) {
  return `${date}T${h}:00${KST_OFFSET}`;
}

// → 2026-07-20T18:00+09:00
Features · 1

주간 그리드로 한눈에 예약

▲ 실제 운영 화면 — 7/6 주

  • 모든 팀 예약을 한 화면에서 — 조회는 비로그인도 가능
  • 생성·수정·취소는 로그인 필수, 본인 예약만 편집
  • 카테고리 — 개인연습은 이름 자동, 기타는 직접 입력
팀마다 색이 달라
누가 언제 쓰는지 바로 보입니다.
Features · 2

임원 운영과 사용 금지 규칙

▲ 매주 월 15:00~17:00 금지 규칙이 자동 표시된 실제 화면

  • /admin 전용 페이지 — 승급·강등으로 인수인계
  • 금지 규칙은 요일·시간대 지정, 무기한 반복
  • 그리드에 회색 블록으로 자동 표시, 겹치면 예약 거부
  • 규칙 조회 실패 시에도 예약 거부 (fail-closed)
Features · 3

공연별 팀 모집 게시판

▲ 실제 게시판 — 세션별 모집 현황("모집중: 기타2")

  • 임원 전용으로 게시판 생성·관리
  • 곡별 모집글 + 세션별 팀원 모집 + 댓글
  • 곡 링크 첨부로 레퍼런스 공유
